ZUGFeRD Properties

Column Description
Property Name The name of the property which is used in the ZUGFeRD JavaScript API.
Description Description of the element as well as possible values.
Required for Describes for which element this element has to be specified to generate the element, if any. If in this field "ZUGFeRD" is written, the ZUGFeRD PDF cannot be generated without this property.
Profile The ZUGFeRD profile in which the ZUGFeRD property is occurring. If in the ZUGFeRD JavaScript API the profile basic has been set, elements that require the profile comfort will not be created. However in comfort, ZUGFeRD properties which require the profile basic are allowed.
Type The type of the ZUGFeRD property. Depending on the type, the value has to meet special criteria.

Property Types

Text

The type Text describes a free text or text content which will not be validated.

Number

The type Number includes amounts and percentages. The decimal mark has to be specified via the optional inputFormat property to ensure that the ZUGFeRD XML is valid. Further information about the inputFormat property can be found in the API documentation.

Date

The Date type requires the current input format for the date string, defined by the inputFormat property, and the output format for the ZUGFeRD document, defined by the scheme property. Further information about these properties can be found in the API documentation. Possible values for the output format are:

102Year, Month and Day: yyyyMMdd
610Year and Month: yyyyMM
616Year und week: yyyyww

Document Type

The Document Type specifies the type of the document through a code. Possible document types are:

84Debit note related to financial adjustments
380Commercial invoice
389Self-billed invoice

Subject Code

The Subject Code type describes the content of a note for electronic validation. Possible codes are:

""General free text
REGRegulatory information
AAKPrice conditions
AAJAdditional conditions of sale/purchase
PMTPayment information

Country Code

The Country Code is a 2 character code for countries. Following are some examples from the ZUGFeRD specification:

DEGermany
FRFrance
ITItaly
CNChina

See

ISO 3166-1 alpha-2

Global ID

The type Global ID specifies the organisation in which the id of the buyer, seller or product is registered. The code of the organisation has to be defined by the scheme property. Further information about this property can be found in die API documentation. Following are some examples from the ZUGFeRD specification:

0021SWIFT (BIC)
0060D-U-N-S Number
0088GS1 Global Location Number (GLN)
0160GS1 Global Trade Item Number (GTIN, EAN)
0177OSCAR (Odette)

See

ISO 6523

3Automated clearing house debit
10In cash
20Cheque

Tax Type

The type Tax indicates which tax type applies. ZUGFeRD defines the following tax types:

VATValue added tax
ZF_INSURANCE_TAXInsurance tax (temporary)
AAJTax on replacement part

Tax Category

The type Tax Category describes which tax category applies. ZUGFeRD defines the following tax categories:

AEVAT Reverse Charge
EExempt from tax
ZZero rated goods
OServices outside scope of tax
ICCode specifying that the VAT rate is levied from the invoicee for Intra-Community supplies.
SValid for regular VAT-rates, e.g. 7 % or 19 % in Germany
